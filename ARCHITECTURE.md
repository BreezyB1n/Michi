# Michi Architecture

## Goal

Michi is a Guide Agent web UI prototype. It turns a user's natural-language intent into a guided, explainable, and verifiable path through a complex product.

The first architecture target is a local browser demo for the Cloudflare Workers primary path:

1. Accept a user intent.
2. Clarify the service type only when needed.
3. Map the intent to a Cloudflare capability.
4. Generate guide steps with purpose and completion checks.
5. Advance through page-state-aware steps.
6. Show recovery steps when the page state blocks progress.

## First Milestone Boundary

The first milestone does not build a real browser extension or automate a live Cloudflare account. It builds the smallest web UI that proves the Guide Loop:

- User Intent parsing
- Capability explanation
- Guide Path generation
- Guide Step display
- simulated Runtime Page Understanding
- Completion Check
- Recovery Step

## System Units

### Web UI

The Web UI is the first user-facing surface. It represents the future In-page Guide Overlay, but runs as a local app for the MVP.

Responsibilities:

- capture User Intent
- show clarifying questions
- show the current Guide Step
- show Step Purpose and Completion Check
- show simulated page-state signals
- show Recovery Steps when blocked

### Guide Agent Core

The Guide Agent Core owns the Guide Loop.

Responsibilities:

- interpret User Intent
- retrieve Site Skill Pack entries
- map intent to Capability
- produce the Guide Path
- choose the next Guide Step based on page state
- request confirmation for Critical Write Actions
- produce Recovery Steps for Blocking States

### Site Skill Pack

The Site Skill Pack is offline product knowledge for a target website. The first pack is Cloudflare-focused.

Responsibilities:

- list relevant Capabilities
- explain product concepts
- define guide path candidates
- mark Critical Write Actions
- define Completion Checks
- describe common Blocking States

### Runtime Page Understanding

The MVP uses simulated page state. A later milestone can replace this with browser extension data from DOM, accessibility tree, page text, or visual recognition.

Responsibilities:

- report current location
- report available actions
- report account or permission signals
- report whether a Completion Check is satisfied
- report Blocking States

## Runtime Flow

1. The user enters a User Intent in the Web UI.
2. The Guide Agent Core checks whether a clarifying question is needed.
3. The Guide Agent Core retrieves Cloudflare Site Skill Pack knowledge.
4. The Guide Agent Core explains the selected Capability.
5. The Guide Agent Core creates a Guide Path.
6. The Web UI displays the current Guide Step.
7. Runtime Page Understanding returns simulated state.
8. The Guide Agent Core runs a Completion Check.
9. The Guide Agent Core advances, asks for confirmation, or emits a Recovery Step.

## Non-Goals

- no live Cloudflare write automation in the first milestone
- no browser extension injection in the first milestone
- no arbitrary script execution generated by the agent
- no full Cloudflare product coverage
- no DNS live-path requirement for the primary demo
