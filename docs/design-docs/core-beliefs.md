# Core Beliefs

## Guide Before Automating

Michi should help users understand and operate complex products. It may highlight, navigate, prefill, or check state, but important write actions stay under user confirmation.

## Intent Before Product Names

Users should not need to know product-specific names before getting value. Michi starts from the user's own words and maps them to product capabilities.

## Every Step Teaches

A Guide Step is not just a click target. Each step must explain what to do, why the step exists, and how progress can be confirmed.

## Page State Is Part Of The Product

Login state, permissions, wrong accounts, changed layouts, and failed validation are not generic errors. They are Blocking States that should become Recovery Steps.

## Demo Scope Must Stay Verifiable

The first Michi milestone should prove a complete Guide Loop before expanding to more sites, more Cloudflare services, or real browser extension automation.
